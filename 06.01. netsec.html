Brute Force vs. Password Spraying

Defending authentication requires understanding the attack pattern. The Malasaña coworking incident proved that simple account lockouts fail against horizontal spraying.

Property Brute Force Password Spraying
Strategy Many passwords vs ONE account ONE password vs MANY accounts
Noise Level High (single-account spikes) Low (each account looks normal)
Defeated By Account lockout, rate limits Aggregate monitoring, MFA
Detection Signal Failed login count PER USER Failed login count PER PASSWORD

Malware Family Tree

Understanding the distinction between payload types is crucial for endpoint defense. Example: The fake BiciMAD APK is a Trojan, not a Virus.

Virus

Needs a host file. Spreads when host is opened.

Worm

Self-replicating. Spreads across network (e.g., WannaCry).

Trojan

Disguised payload. User willingly installs it.

Ransomware

Encrypts files, demands payment. Double extortion.

Modern EDR Stack

How an endpoint agent decides "is this thing bad?" The Lavapiés cybercafé USB example proves why checking hashes against cloud reputation saves networks.

1
Signatures: Match known-bad hashes (useless on zero-day).
2
Heuristics: Score suspicious code structures natively.
3
Behavior: Watch process execution (registry edits, rapid encryption).
4
Cloud Reputation: Has this hash been seen globally?

 ! Deny everything else and LOG
 deny ip any any log
exit

interface GigabitEthernet0/0

Operations & Troubleshooting

The fastest troubleshooters follow a disciplined rhythm, walking down the OSI stack rather than guessing. This section outlines the methodology for isolating network failures and summarizes the final deliverables for the Cyber.SoHo Network Build Challenge.

The Troubleshooting Funnel

1. GATHER

Ask exactly what happened. When did it start? What changed? Read the logs.

2. ISOLATE

Walk the OSI layers. L1 (cable)? L2 (link light)? L3 (ping gateway)? L4 (TCP connect)? Each "yes" eliminates a layer.

3. RESOLVE

Apply the smallest fix. Reboot the smallest possible thing. Have a rollback plan.

4. DOCUMENT

Write the ticket. What was reported, what was found, what was changed. Future-you will be grateful.
