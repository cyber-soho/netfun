◩ Subnetting, Masks & CIDR

This module explores how we divide the 32-bit address space. Subnet masks separate the "network" neighborhood from the "host" house number. You will explore the evolution from rigid Classful networking to modern Classless Inter-Domain Routing (CIDR) and visualize how borrowing bits affects network capacity.

The Subnet Mask

A 32-bit value that splits the address:

  • 1s Network portion (Left side)
  • 0s Host portion (Right side)
255.255.255.0 = /24

"The mask says where the neighbourhood ends and the house number begins. Get it wrong and packets walk into the wrong building."

Classful vs. Classless (CIDR)

Classful (Legacy)

Mask determined strictly by the first octet. Resulted in massive wasted address blocks.

A: 1-126 (/8) B: 128-191 (/16) C: 192-223 (/24)

CIDR & VLSM (Modern)

Mask is whatever you define (e.g., /13, /27). Variable Length Subnet Masking allows fitting subnet size to exact host counts.

Classful = Pre-built suit. VLSM = Tailored suit.

Usable Hosts per Subnet (CIDR /24 to /30)

Notice the exponential drop. Each bit borrowed for the network halves the available hosts.

▣ Address Types & NAT

Not all IP addresses behave the same way. This section outlines the critical differences between routable public addresses, reusable private ranges defined by RFC 1918, and how Network Address Translation (NAT/PAT) bridges the gap between them. It also covers the communication types: Unicast, Multicast, and Broadcast.

Unicast ↑↓

One to one. Default for almost all traffic. Source IP to a single destination IP.

"Order one croqueta."

Multicast ⤑⇉

One to a group. Class D (224.0.0.0/4). Senders send once, only subscribed receivers process it.

"Waiter shouts football score to one table."

Broadcast ⤖⤖

One to everybody. Subnet's last address. Flooded by switches, blocked by routers.

"Bar closes in 5 minutes."

RFC 1918 (Private Ranges)

Reusable across organizations. Not routable on the public internet. Require NAT to talk outward.

10.0.0.0/8 Class A Scale (~16.7M)
172.16.0.0/12 Class B Scale (~1M)
192.168.0.0/16 Class C Scale (~65k)

Other Special Ranges:

  • 127.0.0.0/8 — Loopback
  • 169.254.0.0/16 — APIPA (Link-local)
  • 100.64.0.0/10 — CGNAT (Carrier)

NAT & PAT

The solution to IPv4 exhaustion. Translates private IPs to public IPs.

PAT (NAT-Overload)

Many private IPs share ONE public IP, separated by port numbers (49152–65535).

(Priv_IP:Port) ↔ (Pub_IP:Port)

Static NAT (1:1): One private ↔ one public, fixed.

Dynamic NAT: Pool of public IPs, first-come-first-served.

"PAT is the Hotel Ritz switchboard: hundreds of rooms, one street number, an extension table."

IPv6 — The 128-bit Future

Built to solve IPv4 exhaustion (IANA pool depleted Feb 3, 2011). Offers 2128 addresses, SLAAC, and native multicast (no broadcasts).

Address Scopes

  • ::1 — Loopback
  • fe80::/10 — Link-local
  • fc00::/7 — Unique-local (RFC 4193)
  • Global — Everything else

Shortening Rules

  1. Drop leading zeros in each group.
  2. Replace longest run of zero groups with :: (only once).

⚒ Workshop & Lab Project

Theory must be put into practice. This final module outlines the practical exercises required to complete Project Stage 4: Address the Network. Review the Magic Number method before starting the subnetting tasks.

Magic-Number Method (5 Steps)

  1. Find the interesting octet (mask transitions from 255 to <255).
  2. Compute block size = 256 − value of that mask octet.
  3. Multiples of block size = subnet boundaries.
  4. Network = lower boundary; broadcast = next boundary − 1.
  5. First / last host = network + 1 / broadcast − 1.
