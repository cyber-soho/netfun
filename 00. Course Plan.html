Foundations · How Networks Talk
Tuesday · April 28, 2026 · 4 hours
L1 · Lecture 60 min · 09:00 – 10:00

What is a network — and what carries the data?

A network is two or more devices connected to share data. Devices fall into two camps: hosts (clients and servers) and infrastructure (hubs, switches, routers, APs (Access Points), firewalls). Networks come in geographic flavours:

TypeScopeExample
PAN (Personal Area Network)≤ 10 mBluetooth headset
LAN (Local Area Network)1 buildingOffice Ethernet
MAN (Metropolitan Area Network)1 cityCampus chain
WAN (Wide Area Network)Country / globalThe Internet

Speed & capacity, in plain language

Bandwidth is the link's theoretical capacity (e.g. 1 Gbps). Throughput is what you actually measure on the wire. Goodput is the useful application data once you subtract headers and retransmissions. A 1 Gbps link rarely delivers 1 Gbps of goodput.

bps → kbps → Mbps → Gbps → Tbps 1 → 1 000 → 10⁶ → 10⁹ → 10¹² ASCII reference: 1 byte = 8 bits
Networks exist to move data between hosts. Marketed bandwidth ≠ measured throughput ≠ delivered goodput.

L2 · Lecture 60 min · 11:15 – 12:15

Getting online & the importance of documentation

A host needs four things to reach the internet:

  1. IP address — its unique identifier on the network
  2. Subnet mask — defines which addresses are local
  3. Default gateway — way out to other networks
  4. DNS server — name → address translator

Mobile devices use cellular generations (3G / 4G / 5G) and Wi-Fi. The protocols above the radio layer are mostly the same — IP, TCP, HTTPS. Different physical reach, same packet structure.

Documentation is what separates a working network from a maintainable one. At minimum, a SoHo network needs a logical diagram, a physical diagram, a device inventory, an IP plan, and a change log. If it isn't documented, the next admin (or your future self at 2 a.m.) won't know it exists.

IP + mask + gateway + DNS = online. Miss one and you're isolated, half-online, or "Internet down".

Why "twisted"?

Each pair is twisted to cancel EMI (Electromagnetic Interference). The signal travels differentially — one wire carries +V, the other −V; the receiver subtracts them, so common-mode noise (the same on both wires) cancels out. That's the whole magic.

Distance × speed × cost. Pick two. Fiber wins on distance & speed; copper wins on cost.

Standards bodies & the OSI vs TCP/IP debate

BodyOwns
IEEE (Institute of Electrical and Electronics Engineers)802.3 Ethernet · 802.11 Wi-Fi
IETF (Internet Engineering Task Force)RFCs — TCP/IP, DNS, HTTP, …
ISO (International Organization for Standardization)The OSI 7-layer reference model

OSI ↔ TCP/IP layer mapping

OSI (7 layers) TCP/IP (4 layers) ┌───────────────────────┐ ┌───────────────────────┐ │ 7 Application │ ─┐ │ │ │ 6 Presentation │ ─┼→│ 4 Application │ │ 5 Session │ ─┘ │ │ ├───────────────────────┤ ├───────────────────────┤ │ 4 Transport │ ──→│ 3 Transport │ ├───────────────────────┤ ├───────────────────────┤ │ 3 Network │ ──→│ 2 Internet │ ├───────────────────────┤ ├───────────────────────┤ │ 2 Data Link │ ─┐ │ │ │ 1 Physical │ ─┴→│ 1 Network Access │ └───────────────────────┘ └───────────────────────┘

OSI is the vocabulary; TCP/IP is the reality. When a senior engineer says "that's a Layer-2 issue" they're using OSI numbering against a TCP/IP world.

Layer 1 (Physical) is bits on the wire — voltages, connectors, encoding. Layer 2 (Data Link) wraps those bits into frames with MAC (Media Access Control) addresses and a checksum. Layer 2 has two sub-layers: LLC (Logical Link Control) and MAC.

OSI for vocabulary, TCP/IP for reality. Map between the two when you troubleshoot.

Ethernet & Routing
Tuesday · May 5, 2026 · 4 hours
L1 · Lecture 60 min

The Ethernet frame, encapsulation & the 3-layer design

An Ethernet frame is just bytes in a known order:

┌──────────┬─────┬─────────────┬─────────────┬───────────┬──────────────────┬─────┐ │ Preamble │ SFD │ Dest MAC │ Src MAC │ EtherType │ Payload │ FCS │ │ 7 B │ 1 B │ 6 B │ 6 B │ 2 B │ 46 – 1500 B │ 4 B │ └──────────┴─────┴─────────────┴─────────────┴───────────┴──────────────────┴─────┘ └─── L2 header ────┘ └─── upper layers ──┘ A MAC address is 48 bits, written in hex: 00:1A:2B:3C:4D:5E └── OUI ──┘└── NIC-specific ─┘ (OUI = Organizationally Unique Identifier — the vendor)

Encapsulation, in one diagram

As your data goes down the stack on the sender, each layer wraps it. The receiver unwraps in reverse — that's decapsulation.

Application data │ ▼ + TCP/UDP header → segment / datagram │ ▼ + IP header → packet │ ▼ + Ethernet hdr+FCS → frame │ ▼ encoded as bits → on the wire

Broadcast vs collision domains, and the 3-layer design

Switches shrink collision domains (one per port). Routers bound broadcast domains (one per interface). VLANs (Virtual LANs) let you slice broadcast domains without buying extra routers. The classic hierarchical design has three layers: access (where endpoints plug in), distribution (where policy and routing live), and core (the fast backbone).

Switches break collision domains. Routers (and VLANs) break broadcast domains. Both must scale together.
E1 · Lab 60 min

Wireshark — read your first Ethernet frame

  1. From your Day-2 GNS3 project, start a packet capture on the link between PC1 and the switch.
  2. From PC1, ping PC2.
  3. In Wireshark, select an ICMP echo-request packet.
  4. Identify: destination MAC, source MAC, EtherType (0x0800 = IPv4), the IP header inside, the ICMP payload.
  5. Now ping a destination outside your subnet — observe the destination MAC change to the router's MAC. That's L3 forwarding made visible.
# Useful Wireshark display filters
eth.addr == aa:bb:cc:dd:ee:ff      # By MAC
ip.addr == 192.168.1.10            # By IP
icmp                              # Just ICMP
arp                               # Just ARP
eth.type == 0x0800                # IPv4 frames
L2 · Lecture 60 min

Why we route — and how routers decide

A switch forwards frames within one broadcast domain (Layer 2). A router forwards packets between broadcast domains (Layer 3). The router uses a routing table — a list of "to reach network X, send out interface Y to next-hop Z".

Routes come from three sources:

  1. Directly connected — the router learns these automatically when an interface is up.
  2. Static — typed in by an admin (small networks, edge routes).
  3. Dynamic — learned via routing protocols (OSPF (Open Shortest Path First), EIGRP (Enhanced Interior Gateway Routing Protocol), BGP (Border Gateway Protocol)). 
# A typical "show ip route" output (abridged)
Router> show ip route
Codes: C - connected, S - static, O - OSPF, D - EIGRP, B - BGP

C    192.168.10.0/24 is directly connected, GigabitEthernet0/0
C    192.168.20.0/24 is directly connected, GigabitEthernet0/1
S*   0.0.0.0/0      [1/0] via 203.0.113.1     # default route
Switches keep traffic local. Routers move traffic between networks based on the routing table.

IPv4 anatomy — 32 bits, dotted-decimal, masked

An IPv4 address is 32 bits. Humans write it as four octets in dotted-decimal: 192.168.1.10. Each octet is 8 bits, range 0–255. The address has two parts — network and host — separated by the subnet mask.

Bit values (memorize these): 128 64 32 16 8 4 2 1 │ │ │ │ │ │ │ │ 2⁷ 2⁶ 2⁵ 2⁴ 2³ 2² 2¹ 2⁰ Conversions: 192 → 11000000 168 → 10101000 1 → 00000001 10 → 00001010 192.168.1.10 with /24 mask = 11000000.10101000.00000001.00001010 └────── network (24 bits) ──────┘└host┘

The (now-historical) classes

ClassFirst OctetDefault MaskUsed For
A1 – 126/8Huge nets
B128 – 191/16Mid-size nets
C192 – 223/24Small nets
D224 – 239—Multicast
E240 – 255—Experimental

127.0.0.0/8 is reserved for loopback. Modern networks use CIDR (Classless Inter-Domain Routing), not classful addressing — but the vocabulary survives.

Public vs private (RFC 1918)

RangeMask
10.0.0.0/8
172.16.0.0 – 172.31.255.255/12
192.168.0.0/16

Three delivery models

  • Unicast — one sender, one receiver (almost everything).
  • Multicast — one sender, a defined group (streaming, routing protocols).
  • Broadcast — one sender, every host on the segment (255.255.255.255).
An IP address means nothing without its mask. Always carry the two together: 10.0.0.5/24.

L2 · Lecture 60 min

Static vs dynamic, DHCP, NAT, and why IPv6

DHCP — the DORA exchange

DHCP (Dynamic Host Configuration Protocol) automates client IP configuration. It hands out an IP, mask, gateway, DNS server, and lease time. The full handshake is four messages — Discover, Offer, Request, Acknowledge.

Client Server │ ────── DHCPDISCOVER (broadcast) ──────► │ │ ◄────── DHCPOFFER (here, take 10.0.0.42) │ │ ────── DHCPREQUEST (broadcast: I'll take it!) ► │ │ ◄────── DHCPACK (confirmed, lease = 24 h) │

NAT & PAT

NAT (Network Address Translation) rewrites private addresses into a public one (and back) at the network boundary. PAT (Port Address Translation) is the variant where many private hosts share a single public IP using different source ports. Almost every home router does PAT.

IPv6 — the next 30 years

  • IPv4 has ~4.3 billion addresses → exhausted.
  • IPv6 is 128 bits ≈ 3.4 × 10³⁸ addresses.
  • Built-in features: SLAAC (Stateless Address Autoconfiguration), simpler header (no checksum, no router fragmentation), mandatory IPsec support.
  • Address format: 2001:0db8:85a3:0000:0000:8a2e:0370:7334 → shortened to 2001:db8:85a3::8a2e:370:7334.
DHCP solves "configure 100 PCs". NAT solves "one ISP-given IP, many hosts". IPv6 is what comes next.

The transport layer + the protocols you actually use every day

The transport layer sits between IP — which only knows hosts — and your application, which speaks HTTP (HyperText Transfer Protocol), DNS (Domain Name System), mail, and so on. Two delivery options live here.

TCP vs UDP — two delivery options

PropertyTCPUDP
Connection3-way handshake (SYN, SYN-ACK, ACK)None — fire and forget
ReliabilityAcknowledged + retransmittedNo guarantee
OrderingSequencedOut-of-order possible
Header size20 bytes (minimum)8 bytes
Used byHTTP, HTTPS, SSH, FTP, SMTPDNS, DHCP, VoIP, video, gaming

TCP (Transmission Control Protocol) is a phone call. UDP (User Datagram Protocol) is a postcard.

Ports — the apartment numbers of TCP/IP

A port number selects which application on a host receives a segment. Every TCP/UDP packet carries a source port and a destination port.

ServicePortTransportPurpose
HTTP80TCPPlain web traffic
HTTPS443TCPEncrypted web traffic
SSH (Secure Shell)22TCPSecure remote shell
FTP (File Transfer Protocol)21 / 20TCPFile transfer (control / data)
SMTP (Simple Mail Transfer Protocol)25 / 587TCPSending email
POP3 / IMAP110 / 143TCPReading email
DNS53UDP (TCP for big answers)Name resolution
DHCP (Dynamic Host Configuration Protocol)67 / 68UDPAuto-config

DNS — the phone book of the Internet

DNS turns cyber.soho into 203.0.113.42. The hierarchy is root → top-level domain → second-level → host, queried recursively by your resolver until an authoritative server replies.

your laptop ──► local resolver ──► root (.) ──► .com TLD ──► example.com NS ──► answer (caches) (e.g. 1.1.1.1) (authoritative)

ARP — neighbour discovery on the LAN

ARP (Address Resolution Protocol) answers a single question: "who has IP 192.168.1.1? Tell me your MAC." The reply is cached in the ARP table of every host on the segment.

TCP = phone call (handshake, reliable). UDP = postcard (fast, no guarantee). Ports route packets to the right app on the host. DNS, ARP and DHCP are the silent services that make every other service work.

Wi-Fi standards — the Wi-Fi Alliance naming

Marketing nameIEEE standardBandMax link rate*Year
Wi-Fi 4802.11n2.4 / 5 GHz~600 Mbps2009
Wi-Fi 5802.11ac5 GHz~3.5 Gbps2013
Wi-Fi 6 / 6E802.11ax2.4 / 5 / 6 GHz~9.6 Gbps2019 / 2021
Wi-Fi 7802.11be2.4 / 5 / 6 GHz~46 Gbps2024

*Theoretical link rate — real-world throughput is far lower because of overhead, interference, distance and the number of clients sharing the channel.

CSMA/CA — why wireless is different

Wired Ethernet uses CSMA/CD (Carrier Sense Multiple Access with Collision Detection). Wi-Fi can't detect a collision while transmitting — radios are half-duplex. So it uses CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance): listen, wait a random backoff, transmit, expect an explicit acknowledgement.

Wireless security — WEP → WPA2 → WPA3

  • WEP (Wired Equivalent Privacy) — broken, deprecated, never deploy.
  • WPA2 (Wi-Fi Protected Access 2) — still common; acceptable when patched.
  • WPA3 (Wi-Fi Protected Access 3) — current best practice; mandatory on Wi-Fi 6E and Wi-Fi 7.
  • SSID (Service Set Identifier) hiding is not security — frames still leak the name.
Wi-Fi is shared, half-duplex radio — it cooperates via CSMA/CA. Always pick WPA3 if both sides support it; WPA2 is an acceptable fallback; never WEP and never open.

Threats, attacks & the CIA triad

The CIA triad — what we are protecting

Every security control on every network exists to defend one of three properties: Confidentiality, Integrity, and Availability. Memorise this — every threat below maps to one of those three properties.

Threat catalogue

ThreatWhat it looks likeProperty attacked
Malware (virus, worm, trojan)Self-propagating or hidden codeC / I / A
PhishingFake email or page tricking the userConfidentiality
Man-in-the-MiddleAttacker silently relays + tampers trafficC / I
ARP / DNS spoofingPoisoning name → address bindingsC / I
DoS (Denial of Service)Flood / crash a serviceAvailability
DDoS (Distributed Denial of Service)Flood from a botnetAvailability
RansomwareEncrypts files, demands paymentA (and C if exfiltrated)
Credential reuseAn old leak unlocks a new accountConfidentiality

Defence-in-depth — layered controls

  • Perimeter: firewall rules, segmentation, public services in a DMZ (Demilitarized Zone).
  • Endpoint: patching, anti-virus / EDR (Endpoint Detection and Response), host firewall.
  • Identity: strong + unique passwords, MFA (Multi-Factor Authentication), least privilege.
  • Data: encryption at rest and in transit, backups (the 3-2-1 rule), DLP (Data Loss Prevention).
  • People: security awareness training; the human is always the weakest link.
Security isn't one tool — it's overlapping layers, each one absorbing what the others miss. Map every control you deploy back to C, I, or A.

Reconnaissance — Nmap on your own topology

⚠ Ethics & legality Run Nmap (Network Mapper) only against systems you own or have explicit written permission to test. Scanning third-party hosts without consent is illegal in most jurisdictions, including Canada (Criminal Code §342.1) and Spain (Código Penal §197 bis). In this lab your only target is the GNS3 topology on your own laptop — nothing else.
  • Inside your topology, pick a host and run a host-discovery sweep across its subnet.
  • Run a service-version scan against the HTTP and DNS hosts you built on Day 5.
  • Open the same scan in Zenmap (Nmap's GUI) and compare the visualisations.
  • Document: which ports were open, which service Nmap thought was running, which version it guessed.
# Inside your own GNS3 lab subnet ONLY
nmap -sn 192.168.10.0/24           # Discovery sweep — who is up?
nmap -sV -p 22,53,80,443 192.168.10.5  # Version detection on common ports
nmap -A 192.168.10.5                # Aggressive: OS guess + traceroute + scripts
L2 · Lecture 60 min · 11:00 – 12:00

Firewalls + a structured way to fix things

Firewalls — three flavours

  • Packet filter: rules match L3/L4 fields (source IP, dest IP, port, flags). Stateless.
  • Stateful inspection: tracks connection state — only return traffic of a known flow is allowed back. This is what almost every firewall does today.
  • Next-generation firewall — NGFW (Next-Generation Firewall): adds application identification, identity awareness, intrusion prevention, TLS inspection.

A simple firewall rule, in plain English

ActionSourceDestinationPortComment
ALLOWLAN-Users /24any80, 443, 53Web + DNS outbound
ALLOWanyDMZ web server443Public site
ALLOWAdmin /29any22SSH from jump-box only
DENYanyanyanyDefault deny — last rule

Rule order matters. The first rule that matches wins. Default deny at the bottom is non-negotiable.

Troubleshooting — a structured method

StepWhat you doTools
1 · DefineWhat exactly fails? When? For whom?The user, your eyes
2 · GatherConfigs, logs, recent changesshow running-config, journalctl
3 · LocaliseBottom-up (cable) or top-down (app)?The OSI layers as a checklist
4 · Test L1Cable / link / interface up?Link LEDs, show interface
5 · Test L3Reachability, routingping, traceroute / tracert
6 · Test L4–L7Service listening? Reachable?nslookup, curl -I, nc -vz
7 · Fix & verifyOne change at a time—
8 · DocumentSymptom · cause · fixYour runbook
Firewalls work top-down with a default-deny tail. Troubleshooting works one OSI layer at a time, one change at a time — and the fix is only finished when it's documented.
E2 · Project Stage 6 · Final demo 60 min · 12:00 – 13:00

🚀 Final integration & hand-off

  • Add firewall rules to the edge router using the table above as a starting template.
  • Verify end-to-end: a Wi-Fi client gets DHCP → resolves a name via DNS → fetches the internal HTTP page → reaches the public Internet (simulated).
  • Run a final Nmap sweep from outside the firewall against your DMZ — confirm only port 443 is reachable.
  • Produce a 1-page network document (the deliverable): topology diagram, IP plan table, services list, firewall rule list, one paragraph on what you'd improve next.
  • Demo the topology to the class — five minutes per student.
Hand-off mindset Build it as if you were leaving the company tomorrow and someone else has to maintain it. A network you can't explain on paper is a network nobody can keep alive after you.
