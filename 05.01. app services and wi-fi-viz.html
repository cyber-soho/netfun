TCP vs UDP Protocol Switcher

Transmission Control Protocol

Reliable
  • ✓ Connection-oriented: Requires a three-way handshake (SYN → SYN-ACK → ACK) before data flows.
  • ✓ Reliable & Ordered: Every byte is numbered. Retransmits losses and reorders out-of-order segments.
  • ✓ Graceful Teardown: Closes cleanly with a FIN packet.
  • ! Overhead: 20-byte minimum header and maintains per-connection state.
Use Cases: Web (HTTP/HTTPS), Email, SSH, File Transfer. Use when bytes must all arrive, in order.

User Datagram Protocol

Best-Effort
  • ✓ Connectionless: No handshake, no ACKs, no retransmits. Just shouts and walks away.
  • ✓ Low Overhead: Only an 8-byte header.
  • ✓ Low Latency: Faster delivery because it doesn't wait for acknowledgments.
  • ! Application Reliability: The application must handle its own reliability if needed.
Use Cases: DNS lookups, VoIP, Live Video, Online Games, DHCP. Use when a late packet is worse than a missing one.

Why LAN Segmentation Matters

One flat LAN equals one broadcast domain. Broadcast storms and security incidents scale with hosts. Segmenting via VLANs (Virtual LANs - IEEE 802.1Q) provides Performance, Security, and Manageability.

📦 The "Router" Box

A typical home router is actually four devices in one case:

  • 1. Router: Layer-3 forwarding & NAT
  • 2. Switch: Layer-2 wired ports
  • 3. Access Point: 802.11 radios
  • 4. Firewall: Stateful blocking

🚦 CSMA / CA

Collision Avoidance

  1. Listen: Is channel free?
  2. Wait if busy.
  3. Wait random back-off time.
  4. Transmit data.
  5. Wait for ACK. If no ACK, assume collision, back off, retry.

What WPA3 Fixes

  • SAE: Replaces 4-way handshake, kills offline dictionary attacks.
  • Forward Secrecy: Today's traffic capture doesn't decrypt yesterday's.
  • PMF: Protected Management Frames stops deauth attacks.

Safer Defaults

  • Separate SSID for internet-only guests.
  • Disable WPS (Wi-Fi Protected Setup).
  • Long passphrase (16+ chars random).
  • Use WPA3-Personal or Transition mode.

Lab Exercises

Practical Application and Project Stage 5

Apply your theoretical knowledge through practical exercises. This section details step-by-step instructions for capturing packets in Wireshark and deploying DNS and HTTP services to your Acme network.

Ex 1: Wireshark Captures

60 MIN

Goal: Capture and decode three real packets: ARP, DNS, HTTP GET.

Part A — ARP

Clear cache (arp -d *). Filter: arp. Ping gateway. Verify broadcast "Who has X" and unicast reply.

Part B — DNS

Filter: dns. Run nslookup www.cyber.soho. Identify Query (A record), Response, TTL.

Part C — HTTP GET

Filter: http.request.method == "GET". Browse plain-HTTP site. Follow HTTP Stream. Read headers & status.

🔧 Alternative Tools

tshark (CLI sibling), tcpdump (UNIX sniffer), Ettercap/Bettercap (analysis/MITM), netcat (manual probes), mitmproxy (HTTPS inspection).

Ex 2: Add DNS + HTTP Server

PROJECT STAGE 5
  1. Add VM: Attach Linux VM to SERVERS VLAN (IP 192.168.20.10/24).
  2. Install DNS: apt install dnsmasq. Configure domains to point to .10, .50, .1.
  3. Install HTTP: Run python web server or nginx on port 8080.
  4. Update DHCP: Tell STAFF VLAN router pool to use dns-server 192.168.20.10. Renew client IP.
  5. Verify: From staff client: nslookup www.acme.lan and curl http://www.acme.lan:8080.
