1 The Hardware layer: MAC & Frames

This section breaks down how local communication happens. A MAC address is a hardware-bound identity operating at Layer 2. To send data, it must be packaged into an Ethernet Frame. Interact with the elements below to understand the structure of MAC addresses and the anatomy of a standard IEEE 802.3 frame.

The MAC Address (48-bit)

Key Bits (First Byte):
  • Bit 0: 0 = Unicast, 1 = Multicast/Broadcast
  • Bit 1: 0 = Factory Burn-in, 1 = Locally Administered (Randomized)

2 The Journey: Encapsulation

Data doesn't travel naked. As data moves down the OSI stack, each layer adds its own specific header containing routing and handling instructions. When it moves back up at the destination, headers are stripped. Observe how an application request gets wrapped for transmission.

The OSI Stack (Top Down)

Packet Sniffer View
[Ethernet Header]
Src: AA:BB, Dst: 00:1A
[IP Header]
Src: 192.168.1.10, Dst: 93.184.216.34
[TCP/UDP Header]
Src Port: 51234, Dst Port: 80
[Payload Data]
GET /index.html HTTP/1.1
Rule: Going down adds headers. L3 stays the same end-to-end.

4 Network Design & Routing

For traffic to cross boundaries, networks are built hierarchically and routers use routing tables to find the best path. When multiple sources provide a path, the router trusts the one with the lowest Administrative Distance (AD).

3-Layer Hierarchical Design

Three Sentences to Remember Forever

A switch forwards broadcasts; a router blocks them.
IP gets you there; MAC gets you to the next hop.
Every router only knows the next hop, never the whole path.
